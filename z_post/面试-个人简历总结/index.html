目标检测竞赛

new_idea: 对于不同尺寸bbox, 分别训练不同的三个模型, 小, 中, 大, 由于视角问题, 小的 bbox 一般处在距离摄像头较远的地方, 大的 bbox 一般处在距离摄像头较劲的地方, 这样检测大物体的模型可能就不会错误的将 三轮车, 小的老年代步车认错, 而小模型因为检测的 location 通常处在图片的上面, 所以也不会将处在图片下面的老年代步车错误识别. 这样可以在一定程度上解决这样 FP 样例.

商家招牌: 利用了 attention 机制. 每一个属性分支上都添加 SEBlock. 实验表明对于大部分的类别都有涨点, 最终平均涨点1.x

关于目标检测这边我做了两个竞赛项目, 由于这两个竞赛本身的 metric 计算方式都是只考虑 mAP 而不考虑模型的 inference 耗时, 因此我主要是基于 two-stage 模型进行改进和优化. 根据两个比赛数据集的不同特点, 具体的改进优化方案有一些区别.
先介绍一下 “交通卡口车辆信息识别”, 首先它的数据集大小大约是2w张, 其中 1w 张用于训练, 剩下的用作验证和测试. 数据中的图片规格统一为 1069x500. 比赛要求是要识别出图片中的car, bus, truck 等各种大小的汽车, 同时不能识别图片中的自行车, 电瓶车, 三轮车以及摩托车.
首先, 我的 baseline 方法是用 ResNet-101 作为主干网络, 用 Faster R-CNN 模型作为检测器, 并且采用 paper 中的推荐默认参数进行训练. 最终的 baseline 得分在 78% 左右. 在 baseline 的基础上, 我主要做了以下几点优化.

  1. 首先是确定 anchor box 的尺寸, 从数据集中我观察到, 对于交通卡口来说, 车流量非常大, 车辆之间很密集, 因此, 我使用了更多的 anchor box 数量, 从每个 location 9 个 anchor box 提升到了 12 个, 并用 k-means 聚类算法确定了每个 anchor 初始值. 同时, 我将默认的学习率策略替换成了更加平滑的 consine 学习率衰减策略(Cosine Learning Rate Annealing): $0.5 \times lr \times (cos(\pi \times t / 120) + 1)$, 其中 $t \leq 120$. 并将默认的 NMS 算法替换成了在面对密集物体时鲁棒性更好的 Soft-NMS 算法. 这一步总共带来的 mAP 提升在 4.3% 左右.

  2. 在前一步的基础上, 我使用了一些数据增广技术, 包括水平翻转和 Mixup. 对于 Mixup 的使用我主要分两步进行, 第一步是先根据数据集的 bounding box 标签, 将所有的目标物体裁剪下来, 组成一个新的分类数据集, 这个新数据集中的每张图片都只包含一个物体, 然后利用这个数据集对主干网络(使用 COCO pre-trained 模型)进行分类任务的训练, 训练的过程中会随机的对图片进行 Mixup 操作, Mixup 操作采用的 Beta 分布的 $\alpha$ 和 $\beta$ 参数值均为 0.4 (时间限制没有做充分的消融实验来调试该参数的值, 故而是用 paper 里面的推荐值). 然后, 再用预训练好的模型进行目标检测任务的训练. 在对目标检测数据集进行 Mixup 时, 我只将原始的图片进行融合, 但是不修改标签的值, 这一点和原本的 Mixup 操作有点不同. 主要是因为目标检测任务在融合时, 不仅需要考虑物体的类别, 还需要考虑物体的位置和尺寸, 因此融合起来比较麻烦. 经过水平翻转和 Mixup 的数据增广后, mAP 提升了 1.3% 左右.

  3. 接着, 我在数据集中发现, 由于摄像头的角度问题, 使得图片中的车辆尺寸差异非常大, 近处的车辆比远处的车辆会大好几倍, 因此, 为了解决尺度问题, 同时也为了获得表征能力更强的特征图谱, 我在 backbone 网络中添加了 FPN 来构建包含更多语义的特征金字塔, 以此获得更好的输出. FPN 会在 ResNet 最后一个卷积段之后通过上采样形成 top-down 形式的特征图谱, 然后每一层的特征图谱都会与 ResNet 对应卷积段的特征图谱进行融合, 并且融合后会对每个融合结果进行 $3\times 3$ 的卷积操作, 以此来消除上采样的混叠效果. FPN 的引入使得 mAP 值提升了 1.8% 左右.
  4. 之后, 我用 ResNeXt 替换了 ResNet-101, 并稍微改动了一些超参数的值(anchor 的数量, Mixup 的 $\alpha$ 值), 然后按照相同的策略进行训练, 并将训练后的模型与刚才的模型融合, 最终的 mAP 值提升了不到 1%.
  5. 到这里之后我又尝试了一些改进方法比如 OHEM, Focal Loss 等等, 但是都不太 work. 不 work 的原因我猜测可能主要是因为 OHEM 和 Focal Loss 都是用来解决正负样本不均衡问题的, 而 Faster R-CNN 模型由于 RPN 和 box sampling 的关系, 不太受这些问题的影响. 所以周周转转了几天以后, 我就转而从模型融合的角度进行改进了. 由于一般来说, 模型之间的差异越大, 融合效果就会越好, 因此, 我用了另外两种优化的 two-stage 模型参与融合.
  6. 由于在数据集中, 摄像头的角度是固定的, 而车辆有时候在转弯或者掉头时, 其整体形状会发生一些变化, 因此我选择了 Deformable ConvNet 进行训练, 这个模型的特点是可以对网络中的感受野进行调节, 从而对物体的不同形状具有更好的适应性, 可以捕获到更有效的语义信息. Deformable ConvNet 可以直接在 Faster R-CNN 的框架上应用, 我在 ResNet-101 第一个卷积段 5a, 5b, 5c 网络层上使用了 deformable 卷积层, 同时使用了 deformable RoI pooling 层. 在训练时, Deformable ConvNet 的基本策略和之前的 Faster R-CNN 是相同的, 包括 anchor, lr, soft-nms, 数据增广等等, 主干网络使用的是 ResNet101 + FPN. (没时间训练 ResNeXt).
  7. 另外, 我还使用了从 IoU 角度进行优化的 Cascade R-CNN 模型, 该模型通过一系列的级联过程, 不断的完善 proposals 的假设 IoU 和模型推演时使用的 IoU 阈值的大小, 最终可以生成更精准的候选框, 从而提升模型性能. 同样, Cascade R-CNN 模型也可以在 Faster R-CNN 模型上应用, 并且利用了同样的策略进行训练, 使用的主干网络也是 ResNet101 + FPN.
  8. 在最后融合模型之前, 我对每个模型的预测使用了 Multi-Scale Testing 的策略, 这一步可以为所有模型都带来将近 2% 的 mAP 提升.
  9. 最终, 采用相同策略训练的 DCN 和 Cascade R-CNN 的 mAP 均比原始 Faster R-CNN 模型高出 2~3 个百分点. 于是, 我将前面的两个使用不同 backbone 的 Faster R-CNN 和 Deformable ConvNet 以及 Cascade R-CNN, 总共四个模型进行融合. 融合的策略主要分两步进行: 第一, 对于来自不同模型的检测结果赋予不同的权重, 权重之和为 1. 第二, 将所有模型的预测结果放在一起, 整体执行一遍 Soft-NMS 算法. 阈值为 0.6. 权重的确定方式是, 先给在验证集上具有最大 mAP 的模型分别赋予 0.1~0.9, 步长为 0.2 的权重, 然后剩下的 3 个模型平分剩余的权重. 这样根据最终的得分先确定出第一个模型的最佳权重. 然后, 再用同样的方法依次确定其他模型的权重. 这种方式找出来的不一定是最优的权重分配方式, 但是要比相同权重的融合方式好.

接下来介绍一下 “商家招牌的分类与检测” 目标检测比赛, 这个比赛的任务是识别拍摄的街景图片中的商家招聘, 比如李宁, 肯德基, 华为等等. 这个比赛的数据集有几个很明显的特点, 第一, 它的训练集很少, 只有 4000 多张图片可以训练; 第二, 它的类别数目很多, 总共100个不同的类别; 第三: 类别样本不均衡, 在这100类当中, 不同类别之间的样本数量有时会相差好几倍, 个别样本的数量只有几个或十几个.

这个比赛我使用的检测模型以及一些整体的优化策略和 “交通卡口” 比赛相似, 对于一些细微的变动比如各种参数的尝试和调整就不多说了, 主要说一下针对这个比赛我使用的其他一些改进方法.

  1. 首先是训练集数据量过少的问题, 由于该数据集的训练集数量比较少, 和测试集数量相当, 因此, 更多的数据增广技术是必须的, 除了水平翻转和 Mixup 之外, 我还使用了 random crops, rotations, hue, saturation, exposeure shift 等增广方法, 更多的增广方法对最终的 mAP 有较大的提高, 总共提高了 2%.
  2. 针对正样本类别不均衡问题, 我的解决方法分三步进行, 第一步, 先用整个增广后的数据集进行训练, 并且在采样的时候会对数量较少的样本进行适当的过采样, 保持类别样本数相当均衡, 此时训练好的模型会对整个数据集中的所有类别目标进行检测; 第二步, 将数据集按照类别样本数量的多少划分成三个数据子集(k-means 聚类进行划分), 其中, 样本数量极端少的类别组成一个子集, 样本数量极端多的类别组成一个子集, 剩余的类别组成一个子集; 第三步, 将第一步训练好的模型分别在三个数据子集上进行 fine-tuning, 这样就会得到三个不同的检测模型, 他们只会对各自的子数据集中的类别目标进行检测. 最后, 将第一步得到的一个检测模型和第三步得到的三个检测模型的结果用 Soft-NMS 进行融合. 融合后的模型相比于第一步只使用过采样方法的模型提升了 0.9% 的 mAP 值.
  3. 最后, 我按照相同的策略分别训练了 Deformable ConvNet 和 Cascade R-CNN, 然后将这两个模型的结果和之间的模型结果进行 Soft-NMS 融合. (2.1% 的提升)

Mix up

上式中的 $\lambda \in [0, 1]$ 是从 $Beta(\alpha, \alpha)$ 分布中取得的随机值.

分为两个阶段: 一个是对 backbone 网络中使用 mixup, 另一个是在整体的目标检测网络中使用 mixup.
先根据 ground truth 标签, 将所有的物体裁剪下来, 组成一个新的分类数据集, 数据集每张图片中都只包含一个物体, 然后将所有的图片进行 Mix-Up. Mix-Up 的时候遵循一个原则:(1) 进行 Mix-Up 的两个物体的尺寸差要在一定范围内; (2) 进行 Mix up 的两个物体的类别必须是不同的. 利用该数据集先对 backbone(使用在 COCO 数据集上预训练的模型) 进行 finetuning.
然后对于所有的原始数据集图片, 任选两张进行 mixup(不考虑图片尺寸, 随机选取), mixup 方法是, 根据量张图片的长和宽, 新建一个新的画布, 将两张图片的左上角对齐, 放在新画布的左上角, 然后这个时候我们只需要修改新生成的 image 的宽和高即可, 无需对标签进行修改(包括box和class). 然后将这些新生成的图片随机的放在 mini-batch 的采样集合中进行训练.

和第一名的区别

目标检测: 整体的思路是差不多的, 都是基于 two-stage 模型来做, 可能区别就是在一些细节上吧, 一个是它们尝试了更多种不同的 backbone 网络, 比如 Inception 和 DenseNet, 还有比如数据的增广处理上, 使用了更多的增广技术, 由于它们还用了多卡, 所以 batch 的 size 比我大很多, 加上 BN 的加持和同步通信的优化, 最后就是模型融合的时候, 由于使用了多种不同结构的 backbone 网络, 并且参与融合的数量也非常多, 所以最终的整体效果要比我好很多.

实例分割: 完成度比较低, 只对数据部分, 超参数的设置, 以及学习策略做了一些处理和调整. 最后由于sx就没有关注这个比赛了.

无人机巡检

首先说一下这个项目的具体任务, 就是在电力系统中, 输电线路体系是非常重要的一个环节, 但是由于这些线缆和基建设施常年暴露在室外, 因此难免会由于天气或者其他原因对设施上的组成部件造成一些损坏, 常见的损坏主要有绝缘子自爆, 导线锻炼, 以及设施上的金具固件损坏, 金具固件包括销钉缺销, 防震锤锈蚀, 线夹偏移等等. 另外还有具有塔杆上的异物检测, 主要是鸟巢这一类影响较明显的异物(塑料袋什么的会被风自己吹走, 吹不走的概率低). 我主要参与的就是绝缘子的自爆检测和金具中销钉缺陷检测.
首先是绝缘子的检测, 我们首先是尝试了一下传统的基于图像处理的方法, 由于绝缘子本身的材质比较特殊, 色调信息突出, 因此, 我们就将先将图像转换到 HSV 空间, 然后分别在 H, S通道进行二值化处理, 并将得到的结果进行与运算, 然后得到绝缘子部分的直方图, 然后根据直方图上的峰值来判断绝缘子是否发生自爆. 我们发现这种方法虽然在给定的样本数据集上表现还可以(十几张图片, 2,3张错误), 但是当面对更复杂的场景时, 鲁棒性很差. 因此我们转而使用基于深度学习的算法, 我们的 baseline 是使用 FasterRCNN 对绝缘子进行检测. baseline 方法不论是在 AP 还是在 AR 好, 都超过了传统方法. 然后我们就在此基础上做了一些尝试和改进. 首先是利用 K-means 确定了 anchor 的形状, 由于绝缘子本身是细长型的, 所以 anchor 最终宽高比例也设置的比较大. 其次, 根据检测出来的效果, 我们发现由于绝缘子的缺陷大多都是自爆, 所以到自爆之后, 一串绝缘子还形态上看起来就像是两串离的比较近的绝缘子, 因此很多 badcase 都是把一整串绝缘子检测成了多个绝缘子, 因此我们会在算法后加一步后处理操作, 就是将目标图像检测出的绝缘子进行腐蚀操作, 如果发现腐蚀后的区域有构成了连通区域, 那么我们就将对应的两个 bbox 合并, 这样可以大大降低由于自爆而引起分段的 bad case. 另外, 由于我们得到无人机采集图片像素较高, 大多都是4k的高清图(4608x2592), 因此在实际的项目使用中, 我们不可能直接用这么大的图去训练和测试, 但是如果直接将图像缩小, 那么一定会造成精度上的损失, 当面对小金具时, 损失更大. 为了解决这个问题, 我们提出了多步检测的训练和测试方法, 基本的想法也很简单, 我们分别训练两个不同的模型, 其中一个模型接受的输入是原始图像按比例缩小后的图片, 另一个模型接受的输入是前一个模型检测的坐标结果映射到原始图像上以后, 剪裁出来的局部清晰图片. 我们利用这种方法, 解决了图片尺寸的问题, 同时还在原先的模型上提升了点数. 我们的 backbone 网络使用的是 ShuffleNet, 可以在较快的执行速度下达到我们预期的 AP 和 AR. 当然除了 FasterRCNN 以外, 我们还尝试了很多其他类似的检测模型, 比如由于绝缘子本身细长型的特点, 我们尝试了针对文本检测的 R2CNN 模型, 同时也试了一下分割模型, 比如 MaskRCNN 和 PSPNet 等, 但是效果上都不如 FasterRCNN 好.

销钉缺陷检测, 销钉检测方面基本方法和绝缘子的检测是差不多的, 也是分为两个模型来检测, 从最终的效果上来看也达到了预期的目标, 就是令 AR 达到 90 以上, AP 达到80以上.

为什么没有用比赛时候使用的那些 Trick 和方法, 有些用了, 有些用了效果不是很好, 还有就是因为第一次使用这种和现实场景非常接近的数据集, 实习的大部分时间都花在数据上面了, 包括从项目确立开始的标注方案讨论, 标注结果审查, 很多时候我们需要亲自去一张张的看这些数据, 然后根据看到的结果讨论是不是要修改标注方案, 或者哪些标注是有用的, 哪些标注会带来不好的结果等等. 所以在模型修改这方面做得尝试相对少一些.

车辆内属性识别

这个是刚进组的时候导师交给我的一个练手的项目, 就是对车辆照片中, 驾驶室内的情况进行属性判定, 包括主驾驶和副驾驶是否系安全带, 是否打开遮阳板, 副驾驶是否有人, 车内是否有挂件, 摆件等属性. 我的实现思路也是比较简单, 就是一个 backbone 提取图片特征, 然后是多个并行的属性分支, 每个分支都进行一个三分类操作, 就是判断对每个属性判断 yes, no 以及 unknown. 使用的 loss 是加权后的多属性交叉熵损失, 每个属性都是一个三分类的交叉熵. 在此基础上还在每个属性分支上添加了 attention 模块(BAM, CBAM, SEBlock), 在大多数的属性上均有提点.

EasyInfer

EasyInfer 是一个通用的神经网络前向计算框架, 开发它的主要原因是因为, 在实际生产环境下, 当我们的深度学习模型训练完成后, 我们通常需要将预测模型部署到嵌入式平台上才能正式产生效益, 但是在模型部署过程中, 脱离了训练环境, 就会产生各种依赖问题, 使得部署过程特别麻烦, 另外, 一般的用户对于的神经网络前向计算框架如 TensorRT, NCNN, OpenVINO 等加速引擎的诸多细节都无法一一掌握, 尤其是对于专注于算法提升的算法人员来说, 更是无暇顾及. 基于这一点, 我们开发了 EasyInfer 来帮助用户快速完成模型验证和模型部署工作. EasyInfer 的最大优点就是通用, 这个通用有三层含义, 第一, EasyInfer 的底层支持调用 NCNN, TensorRT 和 OpenVINO 三种不同的推理引擎进行计算, 其中 NCNN 已经作为内置模型默认嵌入在 EasyInfer 当中; 第二, EasyInfer 可以支持多种模型文件格式, 目前可以支持 Caffe, MXNet 和 Onnx 模型, 貌似 PyTorch 的模型支持也已经完成了, 不过这个由于我现在没有权限访问了, 所以不太决定; 第三, EasyInfer 同时提供了 C, C++ 和 Python 三套接口, 并且可以在 Linux, Windows 和 OSX 系统上运行, 未来还计划支持 Android 和 IOS 系统. 当用户需要快速验证模型在部署环境下的性能或效率时, 只需要掌握 EasyInfer 提供的自定义引擎开发接口和自定义模型开发接口, 就可以根据自己的需求快速的完成模型的部署, 同时还能灵活的选择和切换底层的推理引擎, 大大地节省了算法和部署人员的时间成本. EasyInfer 目前已经成功在某银行的人脸识别项目中使用, 同时也为gs内部的门禁系统提供底层技术支持.
在这个项目中, 我主要负责的认为有三个:

  1. 将 MXNet 模型转换成 TensorRT 模型, 为 EasyInfer 的底层引擎添加 MXNet 模型文件支持.
  2. 为 EasyInfer 提供封装完善的 C 接口
  3. 添加 OPENVINO 推理引擎底层支持

下面依次进行介绍, 首先是 MXNet 的模型转换. 在最开始的时候, 由于 TensorRT 仅仅提供了 CaffeParser 的 API 接口, 因此 EasyInfer 在调用底层的 TensorRT 引擎时, 无法使用 MXNet 模型. 对此, 我们最初的解决方案是先将 MXNet 模型转换到 Caffe 模型, 然后再利用 CaffeParser 对模型文件进行解析, 进而生成 TensorRT 接受的 INetworkDefinition 实例对象. 但是这种解决方法的兼容性差, 并且步骤繁琐, 因此, 我们决定重新实现模型转换部分的代码逻辑, 自己实现一个 MXNetParser 来对 MXNet 模型文件进行解析, 我采取的实现思路借助自定义的结构体和 ProtoBuf 来完成模型的转换. 具体来说, 首先根据 MXNet .json 模型文件的字段信息编写存储网络结构和网络参数的结构体. 对于 MXNet 模型文件来说, 它的网络结构描述放置在 .json 结尾的配置文件中, 其神经网络的每一个网络层或者操作在 .json 中都是以一个节点来表示的, 节点与节点之间的连接关系可以通过自身的 inputs 属性决定. 因此, 我们需要存储每个节点的名字, 操作类型, 输入来源以及属性等信息. 然后, 我们对 MXNet 的 .json 和 .params 文件进行解析. 主要方法是顺序读取 .json 文件的节点信息, 然后将节点信息存入到刚刚编写的结构体中, 然后根据节点信息按照字节大小来读取 .params 文件, 将每一个节点的参数值存储在与之对应的结构体当中(通过数组下标对应). 接着, 因为在 MXNet 的.json文件中, 节点是以编号的形式来记录的, 因此, 将节点的信息存储了以后, 为了后续流程能够更加方便的进行, 我们按照节点的 inputs 属性对网络中的节点进行排序, 位置靠前的节点代表网络的较浅层. 到这一步以后, 我们实际上已经拥有了描述网络结构的节点信息以及对应的参数信息. 为了使得代码在设计层面上的抽象程度更高, 我们这里将 MXNet 的网络结构和参数信息都封装在自定义的 protobuf 数据结构中, 这样做的好处有两点, 一是可以统一模型转换的中间数据结构, 方便代码维护和修改, 二是可以在以后进行其他类型的模型转换时进行代码复用, 减少开发成本. 在得到封装好的 protobuf 数据以后, 就可以根据 protobuf 数据中的属性来填充 TensorRT 模型, 填充方法是首先定义一个空的 nvinfer1::INetworkDefinition 对象, 然后根据 protobuf 中的网络层类型调用不同的成员函数. 另外还需要根据之前排好序的节点编号将各个网络层的输入输出串接起来, 使得输入数据可以正确的在网络中执行前向计算, 最终还要特别标记输出层的数据, 以便得到最终的预测结果. 当获取到正确填充后的 nvinfer1::INetworkDefinition 对象实例后, 就可以利用 TensorRT 作为底层引擎来执行 MXnet 模型了.

第二个是 C 接口的封装工作, 由于实际项目需求, 我们需要提供 EasyInfer 的纯 C 语言接口, 主要是将现有的 C++ 接口进行封装, 并对外提供统一完善的 API C 接口.

第三个是添加 OpenVINO 引擎的底层支持, OpenVINO 是英特尔推出的一个视觉推理和神经网络优化的工具包, 可以让开发人员在主流深度学习框架上(TensorFlow, MXNet, Caffe 等)构建和训练人工智能模型, 并将其部署到各种产品中. EasyInfer 对于该引擎也提供了支持. 主要是通过 C++ 代码调用 OpenVINO 官方提供的模型转换工具包, 来将用户的模型文件转换成 OpenVINO 所支持的文件格式:.xml和.bin, 然后再根据用户的输入执行推理计算过程, 并将计算的结果存储到 Buffer 中返回给用户, 由用户自行决定如何使用这些结果.
